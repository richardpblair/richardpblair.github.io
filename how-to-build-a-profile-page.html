Create a Stunninng Profile Page

This project documents the creation of my personal GitHub Pages portfolio—built to answer the inevitable IT interview question: “So… what have you actually built?” Using GitHub Pages, HTML, CSS, and Git, I created a zero-cost, publicly accessible site to showcase hands-on projects, labs, and documentation. Along the way, I broke layouts, fixed commits, learned why version control exists, and proved I can deploy, maintain, and document a live website without setting anything on fire.

CSS HTML GitHub VS Code Git

Objective

The goal was to create a professional page where I could display who I was and what projects I have done to continue to learn and improve.

  • Create a proffesional, publicly accessible portfolio website
  • Spend exactly $0 on hosting
  • Learn how GitHub Pages actually works (not just "click button and hope")
  • Gain hands-on experience with:
    1. GitHub repositories
    2. GitHub Pages
    3. HTML and CSS
    4. Git and version control
  • Build something I can confidently show to employers without saying "please ignore that part"

Tools & Technology

  • GitHub - version control and project hosting
  • GitHub Pages - free static site hosting (thank you, internet gods)
  • HTML & CSS - structure and styling
  • VS Code - where mistakes were made and corrected
  • Git - because "undo" only works so many times
  • Online documentation, tutorial, and AI assistance - for when I stared at the screen and whispered "why?"

Project (Step-by-Step)

I created a GitHub account at https://github.com and spent some time getting familiar with repositories, commits, and branches.

This step was important because GitHub would be doing double duty as:

  • My code repository
  • My public portfolio platform

Also, Github does not care about your feelings - but it does reward consistency.

To use GitHub Pages, the repository name must follow a very specific naming convention:

Example: richardpblair.github.io

This is not a suggestion. This is a rule. GitHub Pages is very polite, but very strict.

I initialized the repository with a README file and confirmed everything was created successfully.

Inside the repository settings. I navigated to:

Rather than starting from a completely blank page, I used the MyPage template from BootstrapMade as a starting point.

Using a pre-built template reflects real-world environments, where you’re often handed existing code and told, “Cool, now make it better.”

During this phase, I:

  • Downloaded and extracted the MyPage template
  • Reviewed the HTML and CSS to understand how the site was structured
  • Customized content, sections, and layout to reflect my experience and projects
  • Removed sections I didn’t need (because less clutter = fewer things to break)
  • Adjusted styling and navigation for clarity and usability

All changes were made locally using VS Code and tracked with Git commits before being pushed to GitHub. Each push automatically redeployed the site using GitHub Pages, which made testing changes fast and mildly addictive.

This step gave me hands-on experience working with an existing codebase — something you do constantly in real IT and development roles.

Helpful = Add the following extensions to view your site as you build it (Found out about this toward the end)

  • Live Preview
  • Live Server

Challenges and lessons learned

  • One missing semicolon can ruin your entire afternoon
  • Small HTML or CSS changes can have surprisingly dramatic results
  • Git commit history is your best friend when something breaks
  • Version control saves you from yourself
  • AI tools are helpful, but understanding why something works is more important than copy-pasting fixes

I am not a web development expert — but I now understand how websites are structured, modified, and deployed without panic.


Final Outcome

  • A live GitHub Pages portfolio website
  • A clean, professional place to showcase projects
  • A repeatable workflow for adding future work
  • Increased confidence using Git, GitHub, HTML, and CSS ( Still a long way to go )

Most importantly, this project allows me to show my work, not just talk about it.

Why This Project Matters

This project demonstrates:

  • Initiative and self-directed learning

For anyone starting in IT or cybersecurity, this is a low-cost, high-impact way to build a professional portfolio and stand out to prospective employers. So go out there. Build things. Break things. Fix them. Learn why they broke — and do it again. That’s how real IT skills are earned.
